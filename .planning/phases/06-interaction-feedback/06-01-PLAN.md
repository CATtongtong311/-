## 任务清单

### Task 1: 全市场股票映射表（模糊匹配确认）

**目标**: 本地缓存所有 A股+港股名称映射，支持中文简称精确匹配和多候选提示。

**详细步骤**:
1. **创建 `src/data/stock_map.py`**
   - `StockMap` 类，管理 `data/stock_map.json`
   - `build_map()` 从 AKShare 拉取全部 A股列表（`stock_zh_a_spot_em`）和港股列表（`stock_hk_spot_em`）
   - 提取字段：code, name, market（sh/sz/hk）
   - 保存为 JSON：`{"名称": [{"code": "600519", "market": "sh", "name": "贵州茅台"}]}`
   - `update()` 方法更新映射表
   - `find(name)` 方法按名称查询，返回候选列表

2. **修改 `src/feishu/message_parser.py`**
   - 在 `MessageParser.parse()` 中文简称匹配后：
     - 如果 `stock_name` 有值但 `stock_code` 为空（ portfolio 未匹配到）：
       - 调用 `StockMap.find(stock_name)` 查询映射表
       - 如果返回唯一结果 → 直接填充 code
       - 如果返回多结果 → `is_valid=False`，`error_hint` 设为候选列表文本（"找到多个匹配：1. 贵州茅台 600519 2. 茅台 0852.HK，请回复数字选择"）
       - 如果无结果 → 保持原错误提示

3. **在 `src/orchestrator.py` 中增加候选选择处理**
   - 新增状态：`pending_selection`（用户有待处理的候选选择）
   - 当用户回复纯数字时，检查是否有待选候选，有则解析为对应股票代码

4. **定时更新映射表**
   - 在 `src/scheduler/jobs.py` 中新增每周日凌晨 03:00 自动更新任务
   - 提供命令行触发：`python -m src.data.stock_map --update`

**验收标准**:
- [ ] 输入"茅台"返回候选列表（600519, 0852.HK）
- [ ] 输入"天赐材料"直接匹配 002709（唯一结果）
- [ ] 映射表可手动更新和自动更新
- [ ] 单元测试覆盖 `StockMap.find()` 各种场景

### Task 2: 飞书卡片按钮 + WebSocket 回调

**目标**: 在诊股卡片中添加交互按钮，通过 WebSocket 接收点击事件。

**详细步骤**:
1. **研究飞书 `card.action.trigger` 事件**
   - 确认 WebSocket 模式下如何接收按钮点击事件
   - 确认事件结构：`event_type`, `open_message_id`, `action.value` 等字段

2. **修改 `src/feishu/gateway.py`**
   - 在 `on_message` 或新增 `on_event` 中处理 `card.action.trigger` 事件
   - 提取 `action.value`（按钮标识，如 `"view_chart:002709"`）
   - 路由到 `orchestrator.handle_card_action(action_value)`

3. **修改 `src/cards/diagnosis_card.py`**
   - 在卡片 `elements` 末尾添加 `action` 区域
   - 按钮定义：
     ```json
     {
       "tag": "action",
       "actions": [
         {"tag": "button", "text": {"tag": "plain_text", "content": "查看分时图"}, "type": "primary", "value": {"action": "view_chart", "symbol": "002709"}},
         {"tag": "button", "text": {"tag": "plain_text", "content": "加入自选"}, "type": "default", "value": {"action": "add_watchlist", "symbol": "002709"}}
       ]
     }
     ```

4. **在 `src/orchestrator.py` 中实现按钮处理**
   - `handle_card_action(action_value)`:
     - `view_chart` → 调用数据接口获取分时图 URL，发送文本消息
     - `add_watchlist` → 将股票添加到 `portfolio.md` 或 portfolio_cache，回复确认消息

5. **处理飞书卡片按钮回调限制**
   - 如果 WebSocket 模式不支持 `card.action.trigger`，回退到文本指令方案
   - 回退检测：启动时检查是否能接收按钮事件，不能则在卡片底部添加文本指令提示

**验收标准**:
- [ ] 诊股卡片显示 [查看分时图] 和 [加入自选] 按钮
- [ ] 点击按钮后机器人正确响应
- [ ] 如果 WebSocket 不支持回调，自动回退到文本指令
- [ ] 按钮响应时间 < 3 秒

### Task 3: 历史诊断记录查询

**目标**: 用户可查询某股票的历史诊断记录。

**详细步骤**:
1. **确认 `DiagnosisHistory` 模型字段**
   - 已有：`symbol`, `name`, `query_time`, `current_price`, `change_pct`, `analysis_text`, `strategy`, `score`, `llm_model`
   - 当前代码中是否已有写入逻辑？检查 `src/analysis/diagnosis.py` 或 `orchestrator.py`

2. **如果缺少写入逻辑，补充记录保存**
   - 在 `orchestrator.py` 诊股完成后，将结果保存到 `DiagnosisHistory`
   - 字段映射：`score` → `diagnosis_result.score`，`strategy` → `diagnosis_result.strategy`

3. **新增查询接口**
   - 在 `src/feishu/message_parser.py` 中增加命令模式：
     - `"历史 002709"`、`"上次诊断 002709"`、`"诊断历史 002709"`
   - 在 `src/orchestrator.py` 中增加 `handle_history_query(symbol)`

4. **创建历史记录卡片 `src/cards/history_card.py`**
   - 查询最近 5 条记录
   - 卡片结构：Header（股票名称+代码），每条记录一行（日期 | 评分 | 策略 | 价格）
   - 如果无记录 → 显示"暂无诊断历史"

**验收标准**:
- [ ] 发送"历史 002709"返回最近 5 次诊断记录卡片
- [ ] 无记录时显示友好提示
- [ ] 诊股完成后自动保存记录
- [ ] 记录包含评分、策略、价格等关键字段

### Task 4: 延迟诊股反馈

**目标**: 诊股 3 天后自动提示用户评价准确性。

**详细步骤**:
1. **新增 `DiagnosisFeedback` 模型（`src/core/models.py`）**
   ```python
   class DiagnosisFeedback(BaseModel):
       diagnosis_id = IntegerField(index=True)  # 关联 DiagnosisHistory.id
       symbol = CharField(max_length=20)
       feedback = CharField(max_length=20)  # "accurate" / "inaccurate" / "pending"
       feedback_time = DateTimeField(null=True)
       created_at = DateTimeField(default=datetime.now)
   ```

2. **修改 `src/core/models.py` 迁移**
   - 确保 `DiagnosisFeedback` 表自动创建
   - 在 `src/core/database.py` 的 `init_db()` 中注册新表

3. **在 `src/orchestrator.py` 诊股完成后调度反馈任务**
   - 保存 `DiagnosisHistory` 后，获取其 id
   - 调用 `scheduler.add_feedback_job(diagnosis_id, symbol, delay_days=3)`
   - delay_days 从 `config/settings.py` 读取（`FEEDBACK_DELAY_DAYS`，默认 3）

4. **在 `src/scheduler/jobs.py` 中新增反馈任务**
   - `send_feedback_prompt(diagnosis_id, symbol)`:
     - 查询该 diagnosis_id 对应的记录
     - 发送飞书文本消息："3天前为您诊股 {name}({symbol})，当时的建议是：{strategy}（评分{score}）。现在回头看，这个判断准不准？回复'准'或'不准'。"
   - 使用 APScheduler 的 `add_job` + `trigger='date'` 设置延迟执行时间

5. **在 `src/feishu/message_parser.py` 中增加反馈命令识别**
   - 识别 "准"、"不准"、"对"、"错" 等关键词
   - 结合上下文：需要关联到最近的反馈提示

6. **在 `src/orchestrator.py` 中处理反馈回复**
   - 收到反馈回复后，更新 `DiagnosisFeedback` 记录
   - 回复确认消息："感谢您的反馈，已记录。"
   - 累计反馈数据用于后续 Prompt 优化（Phase 8 使用）

**验收标准**:
- [ ] 诊股后 3 天自动发送反馈提示
- [ ] 用户回复"准"或"不准"后正确记录
- [ ] 反馈延迟天数可配置
- [ ] 避免重复发送反馈提示（同一诊断只发一次）

## 执行顺序

```
Task 1 (映射表) ──→ Task 2 (按钮回调) ──→ Task 3 (历史查询) ──→ Task 4 (延迟反馈)
    │                    │                    │                    │
    └─ 可独立启动 ───────┴─ 依赖 Task 1 ──────┴─ 依赖 Task 1 ─────┴─ 依赖 Task 3
```

建议顺序：Task 1 → Task 3 → Task 2 → Task 4
（Task 3 历史查询需要映射表支持，Task 4 需要历史记录存在）
