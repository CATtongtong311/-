<objective>
建立多源金融数据获取层，实现本地缓存优先 → Tushare 主源 → AKShare 备用 → iTick 第 3 级备用的自动降级策略，包含数据校验和额度监控。

Purpose: 诊股和晨报均依赖实时金融数据，数据层必须稳定可靠，异常时自动切换备用源；本地缓存减少 API 调用并提升响应速度。
Output:
- DataFetcher: 统一数据接口（个股行情、全球市场、公告新闻），支持 force_refresh 强制刷新
- LocalCache: 本地 JSON 文件缓存，24h TTL，支持 quotes/news/market
- TushareAdapter: 主数据源封装
- AkshareAdapter: 备用数据源封装
- ItickAdapter: 第 3 级备用数据源（免费版 5 次/分钟），含速率限制管理
- DataValidator: 停牌/空值/异常价格校验
- QuotaManager: Tushare 额度监控与降级触发
</objective>

## 关键约束

1. **Tushare 免费额度**: 每日约 100-500 次调用，需实现计数器监控。
2. **AKShare 无需注册**: 完全免费，作为 Tushare 故障/额度耗尽时的自动降级源。
3. **数据校验规则**:
   - 停牌检测: 成交量为 0 或价格无变化
   - 空值检查: OHLC 任一字段为 None/NaN
   - 异常价格: 涨跌幅绝对值 > 20%（A股正常涨跌停 ±10%/±20%）标记为异常但不拒绝
4. **全球市场数据**（DATA-03）:
   - 美股三大指数: 道琼斯、标普500、纳斯达克（Tushare `index_global` 或 AKShare `stock_us_index_spot`）
   - 纳斯达克中国金龙指数: AKShare `stock_hsgt_hist_em` 或相关接口
   - 恒生期货: AKShare `futures_hk_spot`
   - 美元指数: AKShare `fx_spot_quote`
   - 离岸人民币: AKShare `fx_spot_quote` (USDCNH)
   - 10Y 美债收益率: AKShare `bond_us_treasury`
   - **注意**: 具体 AKShare 接口名称以实际文档为准，计划中使用描述性名称，实现时确认。
5. **公告新闻**（DATA-04）:
   - Tushare `major_news` 或 `disclosure` 接口
   - AKShare `stock_notice_report` 或 `stock_news_em` 作为备用
   - 本期仅获取公告标题和摘要，不解析全文
6. **统一返回结构**: 所有 adapter 返回标准化 dict，fetcher 对外提供类型安全的 dataclass。
