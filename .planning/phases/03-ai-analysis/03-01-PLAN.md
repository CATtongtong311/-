# Phase 3 Plan 01: 持仓管理与 AI 大脑

## Objective

实现机器人理解用户持仓并具备智能分析大脑的能力：
1. 从 portfolio.md 读取持仓作为分析上下文
2. 通过 LLM 进行单 Prompt 多维度诊股分析
3. 生成每日投资晨报

## Key Decisions

- **LLM 选型**: 初始使用 Kimi API，用户后续要求切换为 Claude Code CLI (`claude -p`)
- **数据注入方式**: 将实时获取的行情/新闻数据嵌入 Prompt，禁止 LLM 编造数据
- **输出格式**: 诊股要求返回结构化 JSON（评分/策略/价位），晨报返回 Markdown
- **超时策略**: 诊股 600s（10分钟），晨报 300s（5分钟）

## Steps

1. **PortfolioParser 实现**
   - 解析 portfolio.md 的持仓表格（A股 6 位代码、港股代码）
   - 提取成本价、数量、关注板块、备注
   - 实现 ±5% 成本价涨跌警示检测
   - 格式损坏时回退到 SQLite 缓存

2. **LLM Client 实现**
   - 初始：Kimi HTTP API 直连（`api.moonshot.cn`）
   - 后续切换：Claude Code CLI 子进程调用（`claude -p --bare`）
   - 支持自定义超时、UTF-8 编码处理（解决 Windows GBK 问题）

3. **Prompt 模板设计**
   - 诊股 Prompt：四维度分析（技术面/基本面/资金面/情绪面）
   - 要求输出 JSON：score, strategy, support, resistance, stop_loss, reason
   - 晨报 Prompt：三大模块（全球市场/持仓事件/板块前瞻）

4. **DiagnosisAnalyzer 实现**
   - 整合行情数据 + 新闻 + 持仓上下文
   - 调用 LLM 分析 → 解析结构化结果
   - 模块状态跟踪（行情获取/新闻获取/持仓上下文/持仓警示/AI分析/结果解析）

5. **MorningReportGenerator 实现**
   - 整合全球市场数据 + 持仓个股新闻
   - 调用 LLM 生成晨报 Markdown
   - 模块状态跟踪（持仓解析/全球市场数据/个股新闻/AI分析）
