# Phase 4 Plan 01: 集成调度与卡片交付

## Objective

将各组件集成为完整的交互系统：
1. 用户 @机器人后获得结构化诊股卡片（含模块状态汇总）
2. 每日 08:30 自动推送晨报卡片
3. 卡片超限自动裁剪，节假日处理

## Key Decisions

- **诊股发送模式**: 从"即时提示+卡片"改为"汇总模式"——分析完成后统一发送模块状态+结果
- **每日推送**: 用户要求每日执行（不按交易日），SCHED-02 已修改
- **卡片大小**: 预检 25KB 阈值，超限警告但不自动裁剪（依赖内容控制）
- **数据预拉取**: 每晚 19:00 预拉取持仓数据到本地缓存

## Steps

1. **SchedulerManager 实现**
   - APScheduler 后台调度器配置
   - 每日 08:30 晨报推送任务
   - 每晚 19:00 数据预拉取任务
   - 每日 03:00 清理 7 天前晨报记录
   - 支持手动触发（`trigger_now`）

2. **DiagnosisCardBuilder 实现**
   - 飞书 Card JSON 2.0 结构构建
   - Header 颜色对应策略（进攻=红/防御=橙/观望=蓝）
   - 持仓警示显示 🔴 红标
   - 包含评分、策略、支撑压力位、止损线、分析正文

3. **MorningReportCardBuilder 实现**
   - 晨报卡片三模块分栏展示
   - 延迟警告卡片（08:25 未生成完毕时发送）
   - 卡片大小预检

4. **BotOrchestrator 实现**
   - 完整交互链路：飞书消息 → MessageParser 解析 → DiagnosisAnalyzer 分析 → CardBuilder 渲染 → CardSender 发送
   - 诊股结果汇总发送（模块状态 + 分析结果一条消息）
   - 晨报生成和推送
   - 保存诊断历史和晨报记录

5. **数据模型扩展**
   - `MorningReportRecord` 表：记录每次晨报内容、日期、chat_id
   - 数据库初始化包含晨报记录表
